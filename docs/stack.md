1. ``Java 21 선택 이유``

- LTS(Long Term Support) 버전: 가장 최신의 장기 지원 버전으로, 보안 패치와 업데이트를 안정적으로 받을 수 있습니다.

- 가상 스레드(Virtual Threads) 도입: Java 21의 핵심 기능으로, 적은 자원으로도 수만 개의 동시 요청을 효율적으로 처리할 수 있어 서버 성능을 극대화합니다.

- 최신 문법 지원: Record 패턴, Switch 문 확장 등을 통해 가독성 높은 코드를 작성할 수 있습니다.

2. ``Spring Boot 3.5.10 선택 이유``

Java 21 최적화: 최신 버전의 스프링은 Java 17 이상을 필수로 하며, Java 21의 기능을 가장 잘 활용할 수 있도록 설계되었습니다.

현재 가장 안정적인 추천 버전

    - 이유: 3.4 버전의 유지보수가 종료되면서, 현재 보안 패치와 버그 수정이 가장 활발하게 반영되고 있는 안정화된 최신 버전이다.
    - JWT 호환성: 3.4.x에서 쓰던 설정 코드를 그대로 사용할 수 있으며, 내부 라이브러리(Spring Security 6.x)의 보안 취약점이 모두 해결된 상태입니다.


만약 프로젝트를 새로 만드신다면, build.gradle에 다음과 같이 설정하는 것이 가장 안정적입니다.

- Gradle:

```gradle
plugins {
    id 'org.springframework.boot' version '3.5.10'
    id 'io.spring.dependency-management' version '1.1.7'
}
```

3. ``JPA & MyBatis 병행(Hybrid) 선택 이유``

- JPA (생산성): 반복적인 CRUD 작업을 자동화하고 객체 지향적인 설계를 가능하게 하여 개발 속도를 높입니다.

- MyBatis (유연성): 통계 쿼리나 아주 복잡한 조인 등 SQL 튜닝이 직접적으로 필요한 경우, JPA보다 자유로운 SQL 작성이 가능합니다.

결론: 단순 작업은 JPA로 생산성을 높이고, 성능 최적화가 필요한 복잡한 쿼리는 MyBatis를 사용하여 상황에 맞는 최적의 도구를 선택하기 위함입니다.

4. ``PostgreSQL 선택 이유``

- 오픈소스 끝판왕: 라이선스 비용 걱정이 없으면서도 Oracle에 준하는 강력한 기능을 제공합니다.

- JSON 데이터 처리: NoSQL처럼 JSON 형식을 지원하여 비정형 데이터 저장에도 유연합니다.

- 신뢰성: ACID(원자성, 일관성, 고립성, 지속성)를 매우 엄격하게 준수하여 데이터 무결성이 중요한 기업용 프로젝트에 적합합니다.

5. ``Gradle & HikariCP 선택 이유``

- Gradle: XML 기반의 Maven보다 설정이 간결하고, 빌드 속도가 훨씬 빠릅니다.

- [HikariCP](../appendix/hikariCP.md): 스프링 부트의 기본 커넥션 풀로, 가볍고 빠르며 신뢰도가 이미 검증된 업계 표준입니다.
