Spring Boot 2.0 버전부터 HikariCP가 기본 JDBC 커넥션 풀로 채택되었습니다.
그 이전에는 Tomcat DBCP가 주로 사요오디었지만, 현재는 사실상 업계 표준으로 자리 잡았죠

그 이유는 한마디로 요약하자면 ``압도적인 성능과 가벼움`` 때문입니다.

---


1. 압도적인 속도 (Benchmarking)

    HikariCP 개발진은 다른 커넥션 풀(Tomcat DBCP, C3P0, BoneCP 등)과의 성능 비교 결과를 전면에 내세웁니다.
    getConnection() 호출 시 발생하는 오버헤드가 다른 라이브러리에 비해 극도로 낮습니다.

2. 바이트코드 수준의 최적화

    HikariCP는 단순히 코드를 잘 짠 수준을 넘어, 컴파일된 바이트코드 수준에서 최적화를 진행했습니다.

3. 유연한 설계와 작은 버그
    - 마이크로 커널 스타일: 핵심 코드가 매우 슬림합니다. (약 130KB 미만). 코드가 적다는 것은 유지보수가 쉽게 잠재적인 버그가 적다는 뜻입니다.
    - Deaclock 방지: 커넥션 풀이 가득 찼을 때 발생할 수 있는 교착 상태(Deadlock)를 방지하기 위해 계층적인 설계를 도입했습니다.

4. 신뢰성과 안정성

    Spring Boot 팀이 기본 라이브러리를 선정할 때 가장 중요하게 보는 것 중 하나가 커뮤니티의 활성도와 신뢰성입니다.
    HikariCP는 이미 수많은 대규모 서비스에서 안정성이 검증되었고, 설정값이 직관적이며 "꼭 필요한 기능"에만 집중합니다.

---


>  핵심 요약 비교

| 특징 | HikariCP | Tomcat DBCP |
| :--- | :--- | :--- |
| 속도 | 최상(Extreme Fast) | 보통 |
| 코드 크기 | 매우 작음 (Lightweight) | 상대적으로 큼 |
| 설정 | 단순하고 직관적 | 다소 복잡함 |
| Spring Boot | 2.0 부터 기본값 | 1.x버전 기본값 |
